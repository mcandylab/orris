# pino-pretty not installed — server fails to start in development

**Date:** 2026-03-12 12:30
**Files:** backend/package.json
**Severity:** medium

## Problem

`npm run dev` crashes immediately with:
```
Error: unable to determine transport target for "pino-pretty"
```

Fastify is configured to use `pino-pretty` transport in development mode
(`NODE_ENV !== 'production'`), but the package was not in `devDependencies`.

## Root Cause

`pino-pretty` is a peer/optional dependency of `pino`/`fastify` — it is not
automatically installed. The code in `src/index.ts` references it as a named
transport target but it was never explicitly added to the project.

## Solution

Installed `pino-pretty` as a devDependency:
```
npm install --save-dev pino-pretty
```

## Prevention

- When using a named pino transport (e.g. `target: 'pino-pretty'`), always
  add it explicitly to `devDependencies`.
- Review the Fastify logger config block — if it references any transport
  `target`, ensure that package is listed in `package.json`.
